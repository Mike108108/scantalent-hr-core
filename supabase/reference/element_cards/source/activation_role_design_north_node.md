# 0. Pair-level context

## design_north_node / Автоматический вектор развития

Эта позиция показывает, какая gate+line тема может проявляться как повторяющийся вектор развития через опыт, среду и рабочие ситуации: какие задачи, роли, условия и сценарии постепенно расширяют поведение и зрелость человека.

В HR-чтении это помогает понять, какой вектор развития может проявляться через факты поведения, повторяющиеся рабочие ситуации, смену среды, командные сценарии и реальные задачи.

## Связка с personality_north_node

```text
personality_north_node:
“Я могу осознанно видеть эту тему как направление своего роста.”

design_north_node:
“Эта тема проявляется как повторяющийся вектор развития через опыт, среду и рабочие ситуации,
даже если я не всегда сразу это понимаю.”
```

Важно:

```text
North Node-позиция не говорит: “это предназначение человека” или “его финальная профессия”.
Она говорит: “эта gate+line тема выполняет функцию направления развития и будущего вектора в карте”.
```

North Node не заменяет Type, Strategy, Authority, Profile, Centers, Channels, Gates, Lines и whole chart context.

# 2. User-facing Base

# Автоматический вектор развития

## Короткая суть

Эта позиция показывает тему, которая может проявляться как повторяющийся вектор развития через реальные ситуации, рабочую среду, задачи и поведенческий опыт.

Это не скрытое предназначение и не готовая профессия. Это функция конкретной темы в общей рабочей механике человека.

## В работе

В работе такая тема может быть заметна не столько по словам, сколько по тому, в каких ситуациях человек постепенно меняется и через какие задачи расширяет свою роль.

Она может показывать:

```text
какие рабочие сценарии подталкивают человека к развитию;
какая тема повторяется как будущий вектор;
какие условия меняют поведение и роль;
в какой теме опыт постепенно расширяет поведение, роль и зрелость человека;
какие задачи могут вести к более зрелому проявлению.
```

## Здоровое проявление

Человек постепенно входит в развитие через опыт, ситуации и реальные задачи. Даже если он не сразу объясняет направление словами, повторяющиеся рабочие сценарии могут показывать, в какой теме его опыт со временем расширяется.

В здоровом проявлении эта тема помогает человеку расти через опыт, а не через насильственное требование сразу соответствовать будущей роли.

## Искажённое проявление

Искажённое проявление возникает, когда автоматический вектор развития читают как жёсткую судьбу, ярлык или обязанность.

Это может выглядеть как:

```text
среда навешивает на человека будущую роль раньше времени;
повторяющийся сценарий принимают за окончательный вывод;
человека подталкивают к развитию без учёта готовности;
он попадает в задачи, которые развивают, но перегружают;
наблюдаемое направление путают с уже сформированной компетентностью.
```

## Как использовать руководителю

Руководителю полезно наблюдать, какие задачи и среды реально помогают человеку расти.

Практически:

```text
смотреть на повторяющиеся рабочие сценарии;
не делать вывод о будущем пути по одному эпизоду;
давать развитие через практику и постепенное расширение роли;
не навешивать будущую роль как обязанность;
сверять вектор развития с ресурсом, навыками и текущей механикой человека.
```

## Что проверить в реальности

Проверить стоит:

```text
какие ситуации повторяются в рабочем опыте человека;
где он заметно меняется через практику;
какие задачи ведут к росту, а какие просто перегружают;
совпадает ли наблюдаемый вектор с результатами;
не является ли направление следствием давления среды, а не корректного развития.
```

## Важное ограничение

Это не ярлык человека и не готовый HR-вывод. Позиция только уточняет функцию темы; итоговый вывод зависит от всей карты и реального рабочего поведения.

# 3. User-facing Pro

# Позиция активации · Северный узел Дизайна

## Технический смысл

`design_north_node` — это activation_position, где планетарная функция North Node соединяется со стороной Design.

```text
activation_position = planet + side
design_north_node = North Node + Design
```

Эта позиция показывает, как конкретная gate+line тема становится автоматическим вектором развития и темой будущего опыта в карте.

## Planet function

В классической логике Human Design North Node связан с направлением будущего опыта и сценой, через которую тема постепенно раскрывается дальше. В ScanTalent это переводится как рабочий вектор развития конкретной gate+line темы: в каких условиях, задачах и ролях эта тема может становиться более зрелой.

North Node добавляет теме функцию направления:

```text
будущий вектор;
направление развития;
сценарий роста;
переход к следующему этапу;
рабочая перспектива;
развитие роли через опыт;
изменение через среду;
движение к более зрелому проявлению.
```

Важно: North Node не равен “предназначению”, профессии или финальному выводу о человеке. Это направление развития конкретной gate+line темы.

## Side function

Design показывает более автоматическую, телесную и менее осознаваемую сторону проявления.

В этой позиции тема может быть:

```text
заметна через повторяющиеся ситуации;
видна в поведении и рабочем опыте;
сложнее доступна через прямое самоописание;
проявлена через среду, роли и задачи;
понятна со временем через наблюдение;
особенно заметна в изменении поведения и зрелости.
```

Design не означает слабость, проблему или “плохую неосознанность”. Это автоматический слой механики, который нужно наблюдать аккуратно и читать только в контексте всей карты.

## Function in activation composition

Эта позиция уточняет функцию темы в activation composition:

```text
gate + line + planet + side + activation_position
```

`design_north_node` добавляет к gate+line теме функцию автоматического вектора развития.

Пример логики чтения:

```text
gate = какая тема проявляется
line = каким стилем тема проявляется
North Node = тема становится направлением развития
Design = тема проявляется автоматичнее, телеснее, поведенчески
design_north_node = автоматический вектор развития через эту gate+line тему
```

## Contextual reading

Позицию нельзя читать отдельно. Она уточняет тему gate и стиль line, но итоговое значение зависит от:

```text
type
strategy
authority
profile
defined / open centers
full channel или hanging gate
South Node context
реального поведения
наблюдаемого рабочего опыта
условий среды
```

North Node желательно читать вместе с South Node: South Node показывает привычный фон и исходный опыт, North Node — направление развития и будущие условия опыта.

## Correct expression

Корректное проявление возникает, когда автоматический вектор развития раскрывается через подходящий опыт, среду и постепенное расширение роли.

В корректном проявлении человек:

```text
растёт через реальные ситуации;
показывает направление развития поведением, а не обязательно словами;
не обязан сразу объяснять будущий вектор;
получает задачи, которые развивают без разрушительного давления;
двигается в согласии с Type / Strategy / Authority;
со временем может осознать направление через обратную связь и факты.
```

## Not-Self expression

Not-Self проявление возникает, когда Design North Node читается изолированно или когда среда превращает автоматический вектор развития в жёсткое ожидание.

Это может проявляться как:

```text
человека толкают в будущую роль раньше времени;
наблюдаемый вектор превращают в ярлык;
развитие происходит через перегруз, а не через корректный опыт;
повторяющийся сценарий читают как судьбу;
Design-тему требуют объяснить как полностью осознанный план;
AI делает вывод о предназначении без gate, line и whole chart context.
```

## Interpretation limits

Эта карточка описывает не самостоятельное значение North Node и не самостоятельное значение Design, а их совместную функцию внутри activation composition.

Нельзя делать вывод о человеке только по `design_north_node`.

Нельзя трактовать Design как слабость, проблему, тень или отсутствие осознанности. Это автоматическая сторона проявления, которая может быть здоровой и ценной при корректном контексте.

Нельзя трактовать North Node как профессию, предназначение, финальную роль или гарантию карьерного пути. North Node показывает направление развития конкретной gate+line темы, но её качество зависит от всей карты и реального опыта.

Повторяющийся сценарий не является судьбой. Это гипотеза о направлении развития, которую нужно проверять через факты, задачи и динамику поведения.

# 4. Hints

```yaml
talent_hints:
  - помогает увидеть автоматический вектор развития
  - показывает тему будущего опыта через поведение
  - помогает отличить self-report от наблюдаемого роста
  - может указывать на сценарии, которые развивают человека
  - помогает AI отличать привычный фон South Node от будущего вектора North Node

risk_hints:
  - риск принять вектор развития за готовую компетентность
  - риск навесить будущую роль раньше времени
  - риск читать повторяющийся сценарий как судьбу
  - риск перегрузить человека задачами “на рост”
  - риск считать Design-тему слабостью или неосознанной проблемой

management_hints:
  - наблюдать, какие задачи реально развивают человека
  - давать рост через практику и постепенность
  - не делать вывод по одному эпизоду
  - не навешивать будущую роль как обязанность
  - сверять развитие с ресурсом, навыками и Strategy / Authority

environment_hints:
  - среда с постепенным расширением роли
  - задачи, где рост идёт через опыт
  - обратная связь по наблюдаемым изменениям
  - отсутствие давления сразу соответствовать будущей роли
  - возможность видеть развитие через повторяющиеся ситуации

development_hints:
  - учиться замечать повторяющиеся сценарии роста
  - отличать корректное развитие от перегруза
  - переводить наблюдаемый вектор в осознанные шаги
  - не путать будущий опыт с готовой ролью
  - читать North Node вместе с South Node

communication_hints:
  - человеку может быть сложно сразу объяснить свой будущий вектор
  - лучше обсуждать конкретные ситуации, где он менялся через опыт
  - обратную связь давать через наблюдаемые факты
  - не называть автоматический вектор “судьбой” или “предназначением”
  - полезно сверять наблюдаемый рост с результатами
```

# 6. Not-Self / distortion

```yaml
not_self_layers:
  base: >
    Искажённое проявление возникает, когда автоматический вектор развития читают без контекста,
    превращают в ярлык или используют как давление “ты должен туда расти”.

  pro: >
    Not-Self проявление возникает, когда Design North Node читается изолированно от South Node,
    Strategy, Authority, gate, line, channel context и whole chart context. Будущий вектор может быть
    ошибочно принят за судьбу, профессию или уже готовую роль, хотя он является частью механики развития.

  warning_signals:
    - наблюдаемый вектор превращают в жёсткий ярлык
    - человека толкают в будущую роль раньше времени
    - Design-позицию считают скрытым предназначением
    - повторяющийся сценарий принимают за окончательный вывод
    - развитие идёт через перегруз, а не через корректный опыт
    - AI читает design_north_node без South Node и whole chart context

  recovery_conditions:
    - вернуть чтение к gate + line
    - прочитать North Node вместе с South Node
    - проверить full channel / hanging gate context
    - учесть type / strategy / authority
    - отделить будущий вектор от готовой компетентности
    - проверить тему через повторяемое поведение, а не единичный эпизод
    - определить source_role в конкретном HR-слое: primary / supporting / context_only
```

# 7. Limitations

```yaml
limitations:
  - Не делать финальный вывод о человеке только по design_north_node.
  - Не использовать как профессию, должность или HR-роль.
  - Не использовать для fit_score, role-fit, процентов соответствия или решения о найме.

  - Не читать без контекста всей карты.
```

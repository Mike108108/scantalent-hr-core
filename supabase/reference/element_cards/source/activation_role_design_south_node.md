# 2. User-facing Base

# Автоматический привычный фон

## Короткая суть

Эта позиция показывает тему, которая может автоматически проявляться как привычный рабочий фон: повторяющийся сценарий, знакомая среда, исходная настройка поведения или опыт, к которому человек возвращается без долгого осознанного объяснения.

Это не скрытая слабость и не “прошлое, которое мешает”. Это функция конкретной темы в общей рабочей механике человека.

## В работе

В работе такая тема может быть заметна не столько по словам, сколько по повторяющимся ситуациям и поведению.

Она может показывать:

```text
в какие рабочие сценарии человек автоматически попадает;
какая тема повторяется как знакомый фон;
какой способ поведения включается по привычке;
что среда может часто возвращать человеку;
какой исходный опыт лежит под дальнейшим развитием.
```

## Здоровое проявление

Человек естественно использует привычный фон как базу: ему не нужно каждый раз заново осваивать эту тему, и в подходящем контексте она может давать опыт, узнавание и рабочую устойчивость.

В здоровом проявлении эта позиция помогает человеку опираться на знакомый фон и постепенно развиваться дальше.

## Искажённое проявление

Искажённое проявление возникает, когда автоматический привычный фон начинает повторяться без обновления.

Это может выглядеть как:

```text
человек снова попадает в знакомый сценарий, даже если он уже не помогает;
поведение автоматически возвращается к старой реакции;
среда закрепляет привычную роль;
опыт повторяется, но не превращается в развитие;
наблюдаемый повторяющийся паттерн принимают за весь характер человека.
```

## Как использовать руководителю

Руководителю полезно наблюдать, какие сценарии повторяются вокруг человека, но не делать из этого жёсткий ярлык.

Практически:

```text
смотреть на повторяющиеся рабочие ситуации;
отличать привычный фон от текущего потенциала;
не фиксировать человека только в привычной роли;
помогать переводить повторяющийся опыт в развитие;
сверять South Node с North Node, чтобы видеть движение дальше.
```

## Что проверить в реальности

Проверить стоит:

```text
какие рабочие сценарии повторяются;
что человек делает автоматически в знакомой среде;
где привычный фон помогает, а где ограничивает;
какая связь есть с North Node-направлением;
не является ли наблюдаемый фон следствием среды, перегруза или старой роли.
```

## Важное ограничение

Это не ярлык человека и не готовый HR-вывод. Позиция только уточняет функцию темы; итоговый вывод зависит от всей карты и реального рабочего поведения.

South Node — не низший уровень и не ошибка, которую нужно стереть. Это знакомая база опыта, которую важно читать вместе с направлением North Node.

# 3. User-facing Pro

# Позиция активации · Южный узел Дизайна

## Технический смысл

`design_south_node` — это activation_position, где планетарная функция South Node соединяется со стороной Design.

```text
activation_position = planet + side
design_south_node = South Node + Design
```

Эта позиция показывает, как конкретная gate+line тема становится автоматическим привычным фоном, исходной сценой опыта и повторяющимся поведенческим контекстом.

## Planet function

В классической логике Human Design South Node связан с привычной сценой опыта и исходным фоном, из которого раскрывается движение к North Node. В ScanTalent это переводится как рабочая база конкретной gate+line темы: какие сценарии уже знакомы, что автоматически повторяется и из какой темы человек может постепенно расти дальше.

South Node добавляет теме функцию исходного опыта:

```text
привычный фон;
знакомая сцена;
исходный опыт;
повторяющийся сценарий;
рабочая база;
стартовая точка развития;
то, к чему человек может автоматически возвращаться;
тема, из которой начинается движение к North Node.
```

Важно: South Node не равен слабости, регрессу, прошлой жизни, низшему уровню или теме, от которой нужно избавиться. Это знакомая сцена конкретной gate+line темы.

## Side function

Design показывает более автоматическую, телесную и менее осознаваемую сторону проявления.

В этой позиции тема может быть:

```text
заметна через повторяющиеся ситуации;
видна в поведении и реакции на среду;
сложнее доступна через прямое самоописание;
проявлена как привычный паттерн;
понятна со временем через наблюдение;
особенно заметна в условиях нагрузки или знакомой роли.
```

Design не означает слабость, проблему или “плохую неосознанность”. Это автоматический слой механики, который нужно наблюдать аккуратно и читать только в контексте всей карты.

## Function in activation composition

Эта позиция уточняет функцию темы в activation composition:

```text
gate + line + planet + side + activation_position
```

`design_south_node` добавляет к gate+line теме функцию автоматического привычного фона.

Пример логики чтения:

```text
gate = какая тема проявляется
line = каким стилем тема проявляется
South Node = тема становится привычным фоном и исходным опытом
Design = тема проявляется автоматичнее, телеснее, поведенчески
design_south_node = автоматический привычный фон через эту gate+line тему
```

## Contextual reading

Позицию нельзя читать отдельно. Она уточняет тему gate и стиль line, но итоговое значение зависит от:

```text
type
strategy
authority
profile
defined / open centers
full channel или hanging gate
North Node context
наблюдаемого поведения
рабочей среды
повторяющихся сценариев
```

South Node желательно читать вместе с North Node: South Node показывает привычный фон и исходный опыт, North Node — направление развития и будущие условия опыта.

## Correct expression

Корректное проявление возникает, когда автоматический привычный фон используется как база, а не как клетка.

В корректном проявлении человек:

```text
естественно узнаёт знакомую тему через поведение и среду;
может опираться на накопленный опыт;
не обязан сразу объяснять привычный паттерн словами;
не фиксируется только в старой роли;
двигается в согласии с Type / Strategy / Authority;
постепенно переводит знакомую сцену в развитие North Node.
```

## Not-Self expression

Not-Self проявление возникает, когда Design South Node читается изолированно или когда среда закрепляет человека в автоматическом привычном сценарии.

Это может проявляться как:

```text
повторение старого паттерна без осознания;
возврат к знакомой реакции под нагрузкой;
среда снова ставит человека в привычную роль;
наблюдаемый фон превращают в ярлык;
Design-тему требуют объяснить как полностью осознанный выбор;
AI делает вывод о человеке без North Node и whole chart context.
```

## Interpretation limits

Эта карточка описывает не самостоятельное значение South Node и не самостоятельное значение Design, а их совместную функцию внутри activation composition.

Нельзя делать вывод о человеке только по `design_south_node`.

Нельзя трактовать Design как слабость, проблему, тень или отсутствие осознанности. Это автоматическая сторона проявления, которая может быть здоровой и ценной при корректном контексте.

Нельзя трактовать South Node как слабость, прошлую жизнь, регресс, ограничение, низший уровень или тему, от которой нужно избавиться. South Node показывает привычный фон конкретной gate+line темы, но её качество зависит от всей карты, связки с North Node и реального поведения.

# 4. Hints

```yaml
talent_hints:
  - помогает увидеть автоматический привычный фон
  - показывает исходную сцену опыта через поведение
  - помогает отличить self-report от наблюдаемого привычного паттерна
  - может указывать на уже знакомый рабочий сценарий
  - помогает AI отличать привычный фон South Node от будущего вектора North Node

risk_hints:
  - риск повторения старого сценария без обновления
  - риск принять привычный фон за весь характер человека
  - риск закрепить человека в привычной роли
  - риск считать Design-тему слабостью или неосознанной проблемой
  - риск читать South Node без North Node

management_hints:
  - наблюдать повторяющиеся рабочие сценарии
  - использовать привычный фон как стартовую базу
  - не фиксировать человека только в старой роли
  - помогать переводить опыт в развитие
  - читать South Node вместе с North Node

environment_hints:
  - среда, где знакомый опыт признаётся, но не закрепляется как потолок
  - возможность выйти из старой роли постепенно
  - обратная связь по повторяющимся паттернам
  - задачи, где опыт становится базой для роста
  - отсутствие ярлыка “он всегда такой”

development_hints:
  - учиться замечать автоматический привычный паттерн
  - отличать рабочую базу от фиксации
  - переводить старый сценарий в новый уровень опыта
  - не путать повторяемость с судьбой или характером
  - читать South Node вместе с North Node

communication_hints:
  - человеку может быть сложно сразу назвать привычный паттерн словами
  - лучше обсуждать конкретные повторяющиеся ситуации
  - обратную связь давать через наблюдаемые факты
  - не называть автоматический фон слабостью или регрессом
  - полезно сверять наблюдаемый фон с направлением North Node
```

# 6. Not-Self / distortion

```yaml
not_self_layers:
  base: >
    Искажённое проявление возникает, когда автоматический привычный фон читают без контекста,
    закрепляют как ярлык или используют как доказательство, что человек “всегда такой”.

  pro: >
    Not-Self проявление возникает, когда Design South Node читается изолированно от North Node,
    Strategy, Authority, gate, line, channel context и whole chart context. Привычный фон может быть
    ошибочно принят за слабость, регресс или окончательную характеристику, хотя он является частью
    механики исходного опыта.

  warning_signals:
    - наблюдаемый привычный фон превращают в жёсткий ярлык
    - человека снова ставят в старую роль
    - Design-позицию считают слабостью или проблемой
    - повторяющийся сценарий принимают за весь характер человека
    - South Node читают без North Node context
    - AI делает вывод по design_south_node без gate, line и whole chart context

  recovery_conditions:
    - вернуть чтение к gate + line
    - прочитать South Node вместе с North Node
    - проверить full channel / hanging gate context
    - учесть type / strategy / authority
    - отделить привычный фон от финального вывода о человеке
    - проверить тему через повторяемое поведение, а не единичный эпизод
    - определить source_role в конкретном HR-слое: primary / supporting / context_only
```

# 7. Limitations

```yaml
limitations:
  - Не делать финальный вывод о человеке только по design_south_node.
  - Не использовать как профессию, должность или HR-роль.
  - Не использовать для fit_score, role-fit, процентов соответствия или решения о найме.
  - Не читать без gate и line.
  - Не читать без контекста всей карты.
  - Не читать без связки с North Node.
  - Не считать Design проблемной или “неосознанной слабостью”.
  - Не считать South Node слабостью, регрессом или темой, от которой нужно избавиться.
  - Не считать South Node низшим уровнем, который нужно обязательно преодолеть.
  - Не принимать привычный фон за весь характер человека.
  - Не принимать единичное наблюдение за устойчивый привычный паттерн.
  - Не выбирать activation_role напрямую как самостоятельный HR-source без full activation.
```

```yaml
    - обратная связь по повторяющимся паттернам
    - задачи, где опыт становится базой роста
    - отсутствие вывода “он всегда такой”

  key_limitations:
    - не читать изолированно
    - читать вместе с gate + line
    - учитывать whole chart context
    - читать вместе с North Node
    - South Node не равен слабости или регрессу
    - South Node не является низшим уровнем
    - Design не является слабостью

  must_read_with:
    - gate
    - line
    - activation
    - type
    - strategy
    - authority
    - profile
    - center
    - channel, если gate входит в полный канал
    - North Node activation
    - observed behavior when available

  source_role: supporting

  role_in_this_layer: >
    Уточняет функцию темы как автоматического привычного фона внутри конкретного HR-слоя.
```

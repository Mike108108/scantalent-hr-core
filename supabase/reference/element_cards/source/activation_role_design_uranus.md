# 2. User-facing Base

# Автоматическая необычность

## Короткая суть

Эта позиция показывает тему, которая может автоматически проявляться как необычный рабочий паттерн: нестандартная реакция, индивидуальный ритм, неожиданный способ решения, отличие от привычной нормы или естественное обновление способа действия.

Это не признак хаоса, странности, бунтарства или неуправляемости. Это функция конкретной темы в общей рабочей механике человека.

## В работе

Может показывать:

```text
где человек автоматически действует иначе;
какой паттерн выбивается из привычной нормы;
где появляется неожиданный способ решения;
какая тема не укладывается в стандартный процесс;
как человек может обновлять подход без предварительного объяснения.
```

## Здоровое проявление

Человек естественно проявляет необычность в подходящем контексте: замечает иной путь, обновляет привычный способ, видит нестандартное решение и помогает процессу не застревать в старой схеме.

## Искажённое проявление

Искажённое проявление возникает, когда автоматическая необычность читается без контекста или проявляется как резкий разворот, сопротивление норме или поведение, которое команда не успевает понять.

## Как использовать руководителю

```text
смотреть на повторяемый паттерн, а не один эпизод;
просить переводить необычный подход в понятный рабочий эффект;
давать пространство для эксперимента с ясными границами;
не называть нестандартность хаосом без проверки результата;
помогать команде понимать, зачем нужен иной способ.
```

## Что проверить в реальности

```text
где человек автоматически действует иначе;
повторяется ли необычный паттерн в разных ситуациях;
помогает ли он результату или только сбивает процесс;
может ли человек объяснить подход после факта;
не является ли необычность реакцией на стресс, среду или неподходящую роль.
```

## Важное ограничение

Это не ярлык человека и не готовый HR-вывод. Позиция только уточняет функцию темы; итоговый вывод зависит от всей карты и реального рабочего поведения.

Design Uranus не означает хаос, странность, бунтарство, неуправляемость или “неосознанную проблему”. Это автоматический слой уникальности и обновления, который нужно читать через gate, line, центры, каналы и whole chart context.

Нестандартный подход не считается ценным сам по себе. Его нужно проверять через пользу для задачи, команды, процесса или результата.

# 3. User-facing Pro

# Позиция активации · Уран Дизайна

## Технический смысл

`design_uranus` — это activation_position, где планетарная функция Uranus соединяется со стороной Design.

```text
activation_position = planet + side
design_uranus = Uranus + Design
```

Эта позиция показывает, как конкретная gate+line тема становится автоматической необычностью, нестандартным паттерном, индивидуальным способом и темой обновления через поведение и опыт.

## Planet function

В классической логике Human Design Uranus связан с индивидуальностью, необычностью, обновлением и отличием от привычного паттерна. В ScanTalent это переводится как функция рабочей уникальности конкретной gate+line темы: где автоматическое поведение может обновлять, отличаться, нарушать привычный шаблон или предлагать иной способ при наличии реальной пользы для задачи.

```text
нестандартность;
автоматическая необычность;
индивидуальный паттерн;
обновление привычного способа;
неожиданная реакция;
отличие от нормы среды;
иной рабочий ритм;
способ не повторять шаблон;
тема, где возможен другой способ действия.
```

Важно: Uranus не равен гениальности, инновационности, бунту, хаосу, непредсказуемости или проблеме адаптации. Он показывает функцию уникальности конкретной gate+line темы.

## Side function

Design показывает более автоматическую, телесную и менее осознаваемую сторону проявления: тема может быть заметна через поведение, нестандартную реакцию, индивидуальный ритм, изменения, стандарты и нестандартные задачи.

## Function in activation composition

```text
gate + line + planet + side + activation_position
```

`design_uranus` добавляет к gate+line теме функцию автоматической необычности.

## Contextual reading

Читать только через:

```text
type
strategy
authority
profile
defined / open centers
full channel или hanging gate
circuit context
individual circuit context where relevant
collective circuit context where relevant
tribal circuit context where relevant
observed behavior in change / non-standard tasks / renewal
team culture context
stress / overload context
```

`risks_and_distortions` использовать только если Uranus-тема подтверждена повторяющимся поведением, конфликтом со стандартами, непонятным для команды изменением способа или отсутствием практической пользы.

## Correct expression

Человек естественно видит или делает иначе, может обновлять привычный способ через поведение, действует в согласии с Type / Strategy / Authority и переводит нестандартность в пользу для задачи.

## Not-Self expression

Not-Self проявление возникает, когда Design Uranus читается изолированно или когда автоматическая необычность становится резкостью, хаосом, сопротивлением норме или непонятным для команды поведением.

```text
резкий разворот без объяснения;
резкое изменение способа работы без понятного объяснения;
отказ от стандарта без пользы;
поведение, которое команда воспринимает как хаос;
AI делает вывод “человек странный / неуправляемый / бунтарь” без gate, line и whole chart context.
```

## Interpretation limits

Нельзя делать вывод о человеке только по `design_uranus`.

Нельзя трактовать Uranus как гениальность, инновационность, креативность, бунт, хаос, непредсказуемость или проблему адаптации. Нестандартный подход не считается ценным сам по себе; его нужно проверять через пользу для задачи, команды, процесса или результата.

```yaml
talent_hints:
  - автоматическая необычность
  - нестандартный паттерн поведения
  - иной рабочий ритм
  - обновление привычного способа
  - наблюдаемое отличие от нормы среды

  - пространство для нестандартных решений
  - ясные рамки эксперимента
  - открытость к обновлению процессов
  - отсутствие ярлыков “странный” или “хаотичный”
```

# 6. Not-Self / distortion

```yaml
not_self_layers:
  base: >
    Искажённое проявление возникает, когда автоматическую необычность читают без контекста,
    превращают в ярлык или не переводят в понятную рабочую пользу.

  pro: >
    Not-Self проявление возникает, когда Design Uranus читается изолированно
    от Strategy, Authority, gate, line, channel context и whole chart context.
    Автоматическая уникальность может быть ошибочно принята за хаос, бунт,
    странность или неуправляемость, хотя она является частью механики обновления конкретной темы.

  warning_signals:
    - один нестандартный эпизод превращают в ярлык
    - Design-позицию считают слабостью или проблемой
    - необычный подход не объясняется через пользу
    - Uranus читается как гениальность, хаос или бунт
    - команда не понимает практический смысл нестандартности
    - AI делает вывод по design_uranus без gate, line и whole chart context

  recovery_conditions:
    - вернуть чтение к gate + line
    - проверить full channel / hanging gate context
    - учесть type / strategy / authority
    - отделить автоматическую необычность от характера человека
    - проверить тему через повторяемость, контекст и результат
    - определить source_role в конкретном HR-слое: primary / supporting / context_only
```

# 7. Limitations

```yaml
limitations:
  - Не делать финальный вывод о человеке только по design_uranus.
  - Не использовать как профессию, должность или HR-роль.
  - Не использовать для fit_score, role-fit, процентов соответствия или решения о найме.
  - Не использовать для innovation score или creativity score.
  - Не читать без gate и line.
  - Не читать без контекста всей карты.
  - Не считать Design проблемной или “неосознанной слабостью”.
  - Не считать Uranus гениальностью, инновационностью, креативностью или бунтом.
  - Не считать Uranus признаком хаоса, странности или неуправляемости.
  - Не принимать единичную необычную реакцию за устойчивый паттерн.
  - Не считать нестандартность ценной автоматически без пользы для задачи, команды, процесса или результата.
  - Не выбирать activation_role напрямую как самостоятельный HR-source без full activation.
```

```yaml
  - contrast_context: design_uranus vs personality_uranus

      design_uranus показывает автоматическую необычность, которая чаще видна через поведение, реакцию и рабочий ритм.
    why_current_context_is_different: >
      personality_uranus чаще доступен через self-report: человек может сам объяснять свой нестандартный подход.
```

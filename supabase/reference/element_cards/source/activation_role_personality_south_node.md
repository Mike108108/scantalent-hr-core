# 2. User-facing Base

# Осознанный привычный опыт

## Короткая суть

Эта позиция показывает тему, которую человек может осознанно воспринимать как знакомый опыт: то, через что он уже проходил, на что привык опираться и какой рабочий сценарий может считать понятным.

Это не “прошлое, из которого надо уйти”, не слабость и не ограничение само по себе. Это функция конкретной темы в общей рабочей механике человека.

## В работе

В работе такая тема может проявляться как знакомая сцена опыта.

Она может показывать:

```text
какой рабочий сценарий человеку уже понятен;
через какую тему он привык смотреть на задачи;
какой опыт может казаться естественным или самоочевидным;
что он может считать своей привычной базой;
с какой темы часто начинается его развитие дальше.
```

## Здоровое проявление

Человек умеет использовать привычный опыт как базу: не обесценивает его, не застревает в нём и постепенно переводит знакомую тему в более зрелое развитие.

В здоровом проявлении эта позиция помогает человеку опираться на уже накопленный опыт и не начинать каждый раз “с нуля”.

## Искажённое проявление

Искажённое проявление возникает, когда привычный опыт превращается в сценарий, который повторяется без обновления.

Это может выглядеть как:

```text
повторение знакомого способа, даже когда он уже не подходит;
недоверие к новым условиям;
обесценивание будущего развития;
ощущение “я уже знаю, как это работает”, без проверки контекста;
возврат к старому рабочему сценарию под нагрузкой.
```

## Как использовать руководителю

Руководителю полезно понимать, какой опыт человеку уже знаком, но не фиксировать его только в этой зоне.

Практически:

```text
спрашивать, какие сценарии человеку уже понятны;
использовать знакомый опыт как стартовую базу;
не делать из привычной темы потолок развития;
помогать переходить от знакомого опыта к новому шагу;
сверять, где привычный способ помогает, а где начинает ограничивать.
```

## Что проверить в реальности

Проверить стоит:

```text
какой опыт человек сам называет знакомым;
что он делает автоматически, потому что “так уже было”;
где привычный сценарий помогает результату;
где он мешает увидеть новый контекст;
какая связка с North Node показывает направление дальнейшего роста.
```

## Важное ограничение

Это не ярлык человека и не готовый HR-вывод. Позиция только уточняет функцию темы; итоговый вывод зависит от всей карты и реального рабочего поведения.

South Node — не низший уровень и не ошибка, которую нужно стереть. Это знакомая база опыта, которую важно читать вместе с направлением North Node.

# 3. User-facing Pro

# Позиция активации · Южный узел Личности

## Технический смысл

`personality_south_node` — это activation_position, где планетарная функция South Node соединяется со стороной Personality.

```text
activation_position = planet + side
personality_south_node = South Node + Personality
```

Эта позиция показывает, как конкретная gate+line тема становится осознанным привычным опытом, исходной сценой и знакомым фоном, из которого раскрывается дальнейшее развитие.

## Planet function

В классической логике Human Design South Node связан с привычной сценой опыта и исходным фоном, из которого раскрывается движение к North Node. В ScanTalent это переводится как рабочая база конкретной gate+line темы: какой опыт уже знаком, какой сценарий повторяется и через что человек может переходить к дальнейшему развитию.

South Node добавляет теме функцию исходного опыта:

```text
привычная сцена;
знакомый опыт;
исходная база;
освоенный сценарий;
рабочий фон;
стартовая точка развития;
тема, из которой начинается движение дальше;
то, что может казаться естественным или уже понятным.
```

Важно: South Node не равен “прошлому, от которого надо избавиться”, слабости, регрессу или низшему уровню. Это привычная сцена опыта конкретной gate+line темы.

## Side function

Personality показывает более осознаваемую сторону проявления.

В этой позиции тема чаще доступна через self-report:

```text
человек может говорить, что этот опыт ему знаком;
может считать тему понятной или уже освоенной;
может объяснять, почему действует привычным способом;
может осознанно опираться на прошлый опыт;
может видеть в этой теме свою базу.
```

Но Personality не означает полный контроль. Человек может осознавать привычный опыт, но всё равно автоматически возвращаться к нему под нагрузкой или в знакомых условиях.

## Function in activation composition

Эта позиция уточняет функцию темы в activation composition:

```text
gate + line + planet + side + activation_position
```

`personality_south_node` добавляет к gate+line теме функцию осознанного привычного опыта.

Пример логики чтения:

```text
gate = какая тема проявляется
line = каким стилем тема проявляется
South Node = тема становится привычным фоном и исходным опытом
Personality = тема чаще осознаётся и может быть выражена словами
personality_south_node = осознанный привычный опыт через эту gate+line тему
```

## Contextual reading

Позицию нельзя читать отдельно. Она уточняет тему gate и стиль line, но итоговое значение зависит от:

```text
type
strategy
authority
profile
defined / open centers
full channel или hanging gate
North Node context
рабочего опыта
повторяющихся сценариев
условий среды
```

South Node желательно читать вместе с North Node: South Node показывает привычный фон и исходный опыт, North Node — направление развития и будущие условия опыта.

## Correct expression

Корректное проявление возникает, когда человек использует привычный опыт как базу, но не делает из него потолок.

В корректном проявлении человек:

```text
понимает, какой опыт ему знаком;
может опереться на уже освоенную тему;
не обесценивает свою базу;
не фиксируется в старом сценарии;
сверяется со Strategy / Authority;
позволяет привычному опыту стать основой для развития North Node.
```

## Not-Self expression

Not-Self проявление возникает, когда Personality South Node превращается в ментальную привычку: человек объясняет всё уже знакомым опытом и не видит, что контекст изменился.

Это может проявляться как:

```text
фиксация в старом способе;
опора на прошлый опыт без проверки текущей ситуации;
обесценивание новых условий;
повторение знакомого сценария под нагрузкой;
путаница между “мне это знакомо” и “это всё ещё корректно”.
```

## Interpretation limits

Эта карточка описывает не самостоятельное значение South Node и не самостоятельное значение Personality, а их совместную функцию внутри activation composition.

Нельзя делать вывод о человеке только по `personality_south_node`.

Нельзя трактовать South Node как слабость, прошлую жизнь, регресс, ограничение, низший уровень или тему, от которой нужно избавиться. South Node показывает привычный опыт конкретной gate+line темы, но её качество зависит от всей карты, связки с North Node и реального поведения.

# 4. Hints

```yaml
talent_hints:
  - помогает увидеть тему уже знакомого опыта
  - показывает осознанную рабочую базу
  - помогает понять, с какой сцены начинается развитие
  - может указывать на накопленный опыт в теме
  - помогает AI отличать привычный фон South Node от будущего вектора North Node

risk_hints:
  - риск фиксации в знакомом сценарии
  - риск обесценивать новые условия
  - риск путать знакомый опыт с актуальной корректностью
  - риск возвращаться к старому способу под нагрузкой
  - риск принимать self-report “я это знаю” за подтверждённую актуальную компетентность

management_hints:
  - использовать привычный опыт как стартовую базу
  - не фиксировать человека только в знакомой зоне
  - помогать переходить от знакомого сценария к новому шагу
  - проверять, где старый способ всё ещё работает
  - читать South Node вместе с North Node

environment_hints:
  - среда, где опыт признаётся, но не становится потолком
  - задачи, где знакомая база помогает стартовать
  - возможность постепенно выходить в новые условия
  - обратная связь, показывающая, где привычный способ устарел
  - отсутствие давления “сразу отказаться от старого”

development_hints:
  - учиться видеть границу между опытом и фиксацией
  - использовать знакомую тему как основу, а не как финальную точку
  - переводить прошлый опыт в новый уровень применения
  - читать South Node вместе с North Node
  - проверять привычный способ через реальные результаты

communication_hints:
  - человек может уверенно говорить о знакомой теме
  - в интервью полезно спрашивать, где этот опыт уже работал
  - важно проверять, как человек обновляет знакомые подходы
  - обратную связь лучше давать через факты изменения контекста
  - полезно различать уверенность опыта и закрытость к новому
```

# 6. Not-Self / distortion

```yaml
not_self_layers:
  base: >
    Искажённое проявление возникает, когда знакомый опыт становится единственным допустимым сценарием,
    а человек перестаёт замечать, что контекст, роль или условия уже изменились.

  pro: >
    Not-Self проявление возникает, когда Personality South Node читается изолированно
    от North Node, Strategy, Authority, gate, line, channel context и whole chart context.
    Привычная сцена может стать ментальным якорем, который удерживает человека в знакомом способе,
    даже если карта и реальность показывают возможность развития дальше.

  warning_signals:
    - человек слишком быстро говорит “я это уже знаю”
    - знакомый сценарий не проверяется текущими фактами
    - привычный опыт превращается в закрытость к новому
    - South Node читается как слабость или прошлое, от которого надо избавиться
    - AI делает вывод о человеке без North Node context
    - знакомый способ повторяется под нагрузкой, даже когда он уже не помогает

  recovery_conditions:
    - вернуть чтение к gate + line
    - прочитать South Node вместе с North Node
    - проверить full channel / hanging gate context
    - учесть type / strategy / authority
    - отделить опыт от фиксации
    - проверить знакомый сценарий через текущие рабочие факты
    - определить source_role в конкретном HR-слое: primary / supporting / context_only
```

# 7. Limitations

```yaml
limitations:
  - Не делать финальный вывод о человеке только по personality_south_node.
  - Не использовать как профессию, должность или HR-роль.
  - Не использовать для fit_score, role-fit, процентов соответствия или решения о найме.
  - Не читать без gate и line.
  - Не читать без контекста всей карты.
  - Не читать без связки с North Node.
  - Не считать Personality полностью контролируемой стороной.
  - Не считать South Node слабостью, регрессом или темой, от которой нужно избавиться.
  - Не считать South Node низшим уровнем, который нужно обязательно преодолеть.
  - Не путать привычный опыт с текущей компетентностью.
  - Не выбирать activation_role напрямую как самостоятельный HR-source без full activation.
```

contrast_examples:
